Agent memory store (persistent across sessions)
Why memory is dangerous
Memory poisoning persists across sessions — the agent reads this store before every response. An attacker who can write one value can corrupt the agent's behavior forever. OWASP ASI-03

Unlike prompt injection (which resets each turn), poisoned memory is stateful and silent. The agent behaves differently without any new malicious input being sent.
